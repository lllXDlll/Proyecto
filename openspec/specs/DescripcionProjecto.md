# Descripción General del Proyecto

# Sistema de Gestión de Préstamos de Equipos

## Descripción

El Sistema de Gestión de Préstamos de Equipos es una aplicación web diseñada para facilitar la administración y el control del préstamo de equipos dentro de una organización o institución educativa. El sistema centraliza el registro de usuarios, la administración del inventario de equipos y el seguimiento de los préstamos y devoluciones, permitiendo mantener un historial completo de las operaciones realizadas.

La aplicación busca reemplazar los procesos manuales de registro, reduciendo errores, mejorando la disponibilidad de la información y proporcionando una herramienta que permita conocer en tiempo real el estado de cada equipo.

---

# Problema

En muchos laboratorios y organizaciones, el control de préstamos de equipos se realiza mediante registros manuales o herramientas poco especializadas, lo que puede ocasionar problemas como:

* Pérdida de información sobre los préstamos realizados.
* Dificultad para conocer la disponibilidad de los equipos.
* Falta de trazabilidad de las devoluciones.
* Duplicidad en los registros.
* Baja eficiencia en la administración del inventario.

El sistema propuesto busca solucionar estas limitaciones mediante una plataforma centralizada que automatice el proceso de gestión de préstamos.

---

# Objetivo General

Desarrollar una aplicación web que permita administrar de forma eficiente el préstamo de equipos, facilitando el registro de usuarios, la gestión del inventario, el control de préstamos y devoluciones, así como la consulta de información relevante para la administración del sistema.

---

# Objetivos Específicos

* Gestionar el acceso de los usuarios mediante autenticación segura.
* Administrar el inventario de equipos disponibles para préstamo.
* Registrar préstamos y devoluciones de equipos.
* Mantener un historial completo de las operaciones realizadas.
* Proporcionar información actualizada sobre la disponibilidad de los equipos.
* Ofrecer consultas y reportes que apoyen la gestión del inventario.

---

# Usuarios del Sistema

El sistema contempla los siguientes tipos de usuarios:

## Administrador

Responsable de administrar el sistema, gestionar usuarios, administrar el inventario de equipos y consultar la información registrada.

## Encargado

Responsable de registrar préstamos y devoluciones de equipos, así como consultar la disponibilidad del inventario.

---

# Alcance

La primera versión del sistema incluirá las siguientes funcionalidades:

* Autenticación de usuarios.
* Gestión de usuarios.
* Gestión de equipos.
* Registro de préstamos.
* Registro de devoluciones.
* Consulta del historial de préstamos.
* Dashboard con indicadores generales.
* Reportes básicos del sistema.

---

# Beneficios Esperados

La implementación del sistema permitirá:

* Centralizar la información de préstamos.
* Mejorar el control del inventario.
* Reducir errores en los registros.
* Agilizar el proceso de préstamo y devolución.
* Facilitar la consulta de información histórica.
* Incrementar la trazabilidad de las operaciones realizadas.

---

# Metodología de Desarrollo

El proyecto será desarrollado utilizando la metodología ágil **Scrum**, organizando el trabajo en Sprint incrementales. Asimismo, se empleará **Spec-Driven Development (SDD)** mediante OpenSpec para definir y documentar cada funcionalidad antes de su implementación, asegurando la trazabilidad entre requisitos, diseño, desarrollo y pruebas.

---

# Resultado Esperado

Al finalizar el desarrollo se dispondrá de una aplicación web funcional que permita gestionar integralmente el préstamo de equipos, proporcionando una solución organizada, segura y fácil de mantener, preparada para su despliegue en un entorno de producción.
