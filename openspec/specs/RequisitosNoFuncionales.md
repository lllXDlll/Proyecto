# Requisitos No Funcionales

## Objetivo

Este documento describe las características de calidad que deberá cumplir el Sistema de Gestión de Préstamos de Equipos.

---

## RNF-01 - Usabilidad

La interfaz deberá ser intuitiva y permitir realizar las operaciones principales con un número reducido de acciones.

---

## RNF-02 - Rendimiento

Las solicitudes realizadas por los usuarios deberán responder en un tiempo adecuado bajo condiciones normales de operación.

---

## RNF-03 - Seguridad

Las contraseñas deberán almacenarse utilizando algoritmos de cifrado seguros y la autenticación deberá realizarse mediante JSON Web Tokens (JWT).

---

## RNF-04 - Disponibilidad

La aplicación deberá estar disponible durante el período de uso definido para el proyecto, salvo interrupciones programadas por mantenimiento.

---

## RNF-05 - Escalabilidad

La arquitectura deberá permitir incorporar nuevas funcionalidades sin afectar significativamente las existentes.

---

## RNF-06 - Mantenibilidad

El sistema deberá desarrollarse utilizando una arquitectura modular que facilite el mantenimiento y la incorporación de mejoras.

---

## RNF-07 - Compatibilidad

La aplicación deberá ejecutarse correctamente en los navegadores web modernos compatibles con los estándares HTML5.

---

## RNF-08 - Portabilidad

La aplicación deberá poder desplegarse en servicios de alojamiento compatibles con Node.js.

---

## RNF-09 - Calidad del código

El proyecto deberá desarrollarse utilizando TypeScript para favorecer la mantenibilidad y reducir errores durante el desarrollo.

---

## RNF-10 - Pruebas

El sistema deberá contar con pruebas unitarias, de integración y End-to-End que permitan verificar el correcto funcionamiento de los módulos principales.
